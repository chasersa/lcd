# 迷你环境监测仪设计与思考文档

## 一、系统架构图与模块划分

```
+--------------------------+
|       Application        |
|   app.c / app.h / test.c |
|   (主逻辑、界面、按键处理) |
+------------+-------------+
             |
+------------v-------------+
|         Utils            |
|   filter.c/h (滑动平均)   |
|   timer.c   (系统时间)    |
|   delay.c/h (延时函数)    |
+------------+-------------+
             |
+------------v-------------+
|        Drivers           |
|   LED.c/h   - LED控制     |
|   KEY.c/h   - 按键驱动    |

|   RTC.c/h   - 软件RTC     |

|   DHT11.c/h - 温湿度传感器|
+------------+-------------+
             |
+------------v-------------+
|        Hardware          |
|      STM32F407 开发板     |
+--------------------------+
```

### 模块功能说明

- **Application层**：实现核心业务逻辑，包括每秒读取传感器、更新LCD显示、处理按键切换界面、控制LED状态、通过串口上报JSON数据。
- **Utils层**：提供通用的工具函数，如滑动平均滤波（`filter.c`）、系统时间基准（`timer.c`）。
- **Drivers层**：对硬件外设的底层驱动封装，包括LED、按键、串口、RTC、LCD和DHT11传感器。各驱动提供简洁的初始化与操作接口，上层无需关心硬件细节。
- **硬件层**：基于STM32F407微控制器。

## 二、三处对AI生成代码的修改与优化

### 1. RTC初始化函数重命名
**问题**：原AI生成的`rtc.h`中声明了`u8 RTC_Init(void)`，与STM32标准外设库中的`ErrorStatus RTC_Init(RTC_InitTypeDef*)`函数名冲突，导致编译错误。
**修改**：将驱动中的初始化函数更名为`MyRTC_Init`，并在所有调用处同步修改。
**原因**：避免与标准库命名冲突，确保项目可正常编译，同时保留软件RTC的独立功能。

### 2. 按键读取方式调整
**问题**：原代码使用`KEY_S1_READ()`宏读取按键状态，但该宏在`key.h`中定义为`GPIO_ReadInputDataBit(GPIOF, GPIO_Pin_8)`，由于头文件包含顺序问题，导致链接时找不到宏展开后的函数。
**修改**：在`test.c`中直接使用`GPIO_ReadInputDataBit(GPIOF, GPIO_Pin_8)`读取按键，并显式包含`stm32f4xx_gpio.h`。
**原因**：确保按键扫描函数在任何编译环境下都能正确获取引脚电平，避免隐式声明和链接错误。

### 3. 变量声明位置适配C89标准
**问题**：在`filter.c`和`DHT11.c`中，原AI代码将变量声明放在可执行语句之后（如`for(uint8_t i=0;...`），但Keil编译器默认采用C89模式，要求所有变量声明必须在函数或块的开头。
**修改**：将循环变量`i`、`j`等提前声明到函数或语句块的开头。
**原因**：保证代码符合Keil编译器的语法要求，消除编译错误，同时保持代码的可移植性。

## 三、开发过程中遇到的关键问题及解决思路

### 问题1：多重定义 `SysTick_Handler`
**现象**：链接时提示`Symbol SysTick_Handler multiply defined (by timer.o and stm32f4xx_it.o)`。
**分析**：我们新增的`timer.c`中包含了`SysTick_Handler`实现，而工程原有的`stm32f4xx_it.c`中也存在该中断服务函数。
**解决思路**：
- 移除`timer.c`中的`SysTick_Handler`定义，仅保留`GetTick`函数。
- 在`stm32f4xx_it.c`的`SysTick_Handler`中添加`gSystemTick++`，同时保留原有的`TimingDelay_Decrement()`以支持`delay_ms`功能。
- 将`gSystemTick`声明为全局变量，并在`timer.c`中定义，供其他模块通过`GetTick`访问。

### 问题2：链接时大量未定义符号（如`LCD_Clear`、`UART3_Configuration`等）
**现象**：编译通过但链接失败，提示找不到驱动函数。
**分析**：工程中文件路径未正确包含。
**解决思路**：
- 确认头文件包含路径已指向正确的驱动目录。
- 对于`UART3_Configuration`，确保`USART.c`中已实现该函数，且函数名与调用一致（原驱动中为`UART3_Configuration`，而非`uart_init`）。
通过以上调整，所有链接错误得以解决，系统功能正常实现。
